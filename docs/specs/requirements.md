# 要件定義: Auto Manual Dictionary

## 目的

大量の自動車整備マニュアルHTML（日英、非対称、ファイル名不一致）から、検索/RAG/翻訳補助に使える多言語用語辞書を Python だけで半自動生成・育成する。

## 背景課題

- 日英マニュアルが必ずしも 1:1 対応しない。
- ファイル名やURLが一致しない。
- 章分割・ページ分割・仕向地・版が異なる可能性がある。
- 単純な日英ペア辞書では文脈違いに弱い。
- 会社環境では Python しか動かない。

## 成果物

1. Python CLI
2. SQLiteデータベース
3. JSONL/CSVエクスポート
4. レビューキュー出力
5. 評価レポート
6. テスト一式

## スコープ内

- ローカルHTMLディレクトリの読み込み
- HTMLから本文・見出し・表・リンク・画像参照・数値アンカーを抽出
- ページ特徴量の作成
- block/section/evidence中心の日英断片候補対応づけ
- ページ対応は補助特徴量として扱い、1:1前提にしない
- 用語候補抽出
- 概念ID中心の辞書候補管理
- 証拠追加による確信度更新
- `candidate` / `review_ready` / `confirmed` / `blocked` の状態管理
- レビュー対象のCSV/JSONL出力
- confirmed辞書の検索クエリ展開用エクスポート

## スコープ外（初期）

- Web UI
- クラウドDB
- OpenSearch/Elasticsearch連携
- LLM API必須処理
- 完全自動承認
- PDF/OCR処理

## 非機能要件

- Python 3.10+ で動作
- pip でPythonパッケージを導入できる前提
- SQLiteは標準 `sqlite3` で利用できる
- 外部DBサーバーなしで処理可能
- 入力HTMLを改変しない
- 処理結果は再現可能
- すべての候補に根拠を保持する
- confidenceは内訳とversionを保持し、説明可能にする
- 安全系用語は人間承認なしに confirmed にしない
- confirmed と search/RAG safe は分離する
- 機密HTMLはローカル処理し、外部API送信を必須にしない
- ログに本文全文や機密HTML断片を大量出力しない

## 主要ユーザー

- 整備マニュアル検索/RAGを作る開発者
- 日英整備用語辞書をレビューする専門家
- 検索品質を評価する担当者

## 成功条件

PoC段階:

- 100〜500 HTMLページを取り込める
- ページ特徴量をSQLiteに保存できる
- 100〜300件の用語候補を生成できる
- 閾値超え候補をレビューキューに出力できる
- confirmed辞書でBM25検索の再現率または正解ページ上位率が改善する

## 用語

- concept: 日英表現を束ねる概念。例 `SYMPTOM_ENGINE_NO_START`
- term: 日本語または英語の表現。例 `始動不良`, `engine does not start`
- evidence: 用語対応の根拠。文脈、ファイル、アンカー、スコアを含む
- review_ready: 閾値を超え、人間レビュー対象になった状態
